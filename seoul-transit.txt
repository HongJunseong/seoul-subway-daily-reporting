프로젝트 이름 (예시)

“Seoul Transit Lakehouse & LLM Auto-Report”

한 줄 요약

서울시 대중교통(지하철/버스)의 실시간 + 집계 API를 수집해서
Delta Lake 기반 Lakehouse + DW/DM를 만들고,
매일 LLM이 자동으로 혼잡/이상 패턴을 요약 리포트해주는 시스템.

📡 어떤 실제 데이터를 쓸 수 있냐?

전부 실제 공공데이터 / OpenAPI라 “데이터 직접 생성”이 아니라 실제 서비스 데이터야.

🧊 Delta Lake / AWS / LLM을 어디에 얹을까?
🔹 Storage & Lakehouse

S3 + Delta Lake

/bronze/bus_arrival/ : API Raw JSON 그대로 Delta

/silver/fact_bus_arrival/ : 정제된 버스 도착 Fact

/silver/fact_subway_passenger/ : 승하차 Fact

/gold/dm_* : DM 테이블들

🔹 Glue / Athena

Glue Catalog로 Delta 테이블 등록

Athena로 바로 쿼리 & 분석

🔹 Spark (Delta)

로컬 → 나중에 EMR / Glue Spark Job으로 확장

Batch ETL + (원하면) Structured Streaming 도입

🔹 Airflow

DAG 예시:

fetch_bus_arrival_raw (5분/10분 간격 API 호출 → S3 Bronze)

fetch_subway_passenger_raw (하루 1회 배치)

build_silver_facts

build_gold_marts

generate_llm_report (Gold 기준 요약 생성 → Slack/메일/Webhook)

🔹 LLM 자동화

Gold의 dm_daily_transit_summary에서 집계된 수치 뽑고,

Prompt 예시:

“다음 지표를 기반으로 서울 대중교통 상황 일일 리포트 작성해줘.

전체 승차 인원: ...

전일 대비 증감: ...

가장 혼잡한 역 TOP3: ...

가장 지연이 잦은 버스 노선 TOP3: ...

인사이트/원인/추가로 모니터링할 포인트까지 포함해줘.”

→ 이걸 Airflow에서 매일 자동으로 돌리면
**“AI/LLM 기반 데이터 업무 자동화 경험”**이 딱 된다.
